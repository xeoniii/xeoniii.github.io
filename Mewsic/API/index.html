Mewsic Plugin API

v0.9.4 Documentation

Welcome to the Mewsic Plugin API! Mewsic exposes a global window.Mewsic object to allow plugins to interact with the application state, control playback, query the library, interact with the UI, and listen to application events.

Accessing the API

The API is globally available on the window object:

const mewsic = window.Mewsic;

1. Application Information

Mewsic.version

Returns the current version of the Mewsic application as a string.

console.log(window.Mewsic.version); // e.g. "0.9.4"

2. Player Controls

The Mewsic.player object provides properties and methods to control and inspect playback state.

Properties

  • currentTrack: Track | null
    Returns the currently playing track object.
  • isPlaying: boolean
    Returns true if media is currently playing.
  • volume: number
    Returns the current volume level (0.0 to 1.0).
  • queue: Track[]
    Returns an array of tracks currently in the queue.
  • currentTime: number
    Returns the current playback time in seconds.
  • shuffleEnabled: boolean
    Returns true if shuffle mode is enabled.
  • repeatMode: "off" | "one" | "all"
    Returns the current repeat mode state.
  • currentPlaylistName: string | null
    Returns the name of the playlist currently playing, or null if the playback source is not a playlist.

Methods

  • play()
    Resumes playback.
  • pause()
    Pauses playback.
  • togglePlay()
    Toggles between play and pause states.
  • next()
    Skips to the next track in the queue.
  • prev()
    Returns to the previous track in the queue.
  • seek(time: number)
    Seeks to the specified time in seconds within the current track.
  • setVolume(volume: number)
    Sets the volume level. volume should be a number between 0.0 and 1.0.
  • toggleShuffle()
    Toggles the shuffle state.
  • setRepeatMode(mode: "off" | "one" | "all")
    Sets the repeat mode explicitly.
  • playTrack(trackId: string)
    Immediately plays a track from the library given its trackId. Replaces the current queue with the library queue.
  • setQueue(tracks: Track[], startIndex = 0)
    Overrides the current queue with a given array of Track objects, starting playback at the specified startIndex.
  • addToQueue(track: Track)
    Appends a single Track object to the end of the current queue.
  • clearQueue()
    Clears the playback queue and stops playing.

3. Library Access

The Mewsic.library object gives read-only access to user tracks and playlists, along with helper methods.

Properties

  • tracks: Track[]
    Returns all tracks available in the user's library.
  • playlists: Playlist[]
    Returns all playlists available in the user's library.

Methods

  • playPlaylist(playlistId: string)
    Immediately starts playing the specified playlist by ID, updating the queue.
  • search(query: string): Track[]
    Searches the library by track title, artist, or album and returns an array of matching Track objects.

4. UI Interactions

The Mewsic.ui object allows you to control the application's appearance and prompt feedback to the user.

Properties

  • activeView: string
    Returns the currently active main view.

Methods

  • addNotification(message: string, type: "info" | "success" | "error" = "info", duration = 5000, title?: string)
    Displays a toast notification to the user.
  • setTheme(theme: "dark" | "light")
    Sets the application's visual theme.
  • setAccentColor(color: string)
    Changes the global application accent color (can be a standard CSS color value).
  • setView(view: string)
    Changes the main active view of the application (e.g. 'library', 'settings').

5. Event System

The Mewsic.events object allows plugins to listen and react to application state changes.

Methods

  • on(event: string, callback: Function)
    Subscribes a callback function to the specified event.
  • off(event: string, callback: Function)
    Unsubscribes a previously registered callback from the specified event.

Available Events

  • track_changed
    Fired when the current track changes.
    Payload: Track | null
  • playback_state_changed
    Fired when playback is paused or resumed.
    Payload: boolean (isPlaying)
  • volume_changed
    Fired when the volume level changes.
    Payload: number (volume)
  • time_changed
    Fired when the playback time updates.
    Payload: number (currentTime)
  • shuffle_changed
    Fired when shuffle mode is toggled.
    Payload: boolean
  • repeat_changed
    Fired when repeat mode changes.
    Payload: "off" | "one" | "all"
  • view_changed
    Fired when the active UI view changes.
    Payload: string (activeView)
  • playlist_changed
    Fired when the active playback queue source changes to a new playlist.
    Payload: string | null (playlistName)

Example Usage

Here is a basic example of how a plugin might listen to track changes and display a notification:

// Ensure Mewsic API is available
if (window.Mewsic) {
  // Listen for track changes
  window.Mewsic.events.on("track_changed", (track) => {
    if (track) {
      window.Mewsic.ui.addNotification(
        `Now Playing: ${track.title} by ${track.artist}`,
        "info",
        3000,
        "Plugin Alert"
      );
    }
  });

  // Example: Play the next track when a specific action occurs
  const skipToNext = () => {
    window.Mewsic.player.next();
  };
}
